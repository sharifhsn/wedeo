[2026-03-23] | Tracing was gated behind compile-time features (#[cfg(feature = "tracing")], tracing-detail, optional tracing-subscriber). This made traces unavailable in release builds and required --features tracing everywhere. | NEVER use compile-time feature gates for tracing. Libraries depend on `tracing` unconditionally, binaries always init `tracing-subscriber`. Runtime control via RUST_LOG only. The tracing crate is near-zero-cost when no subscriber is active (single atomic load).

[2026-03-24] | `cargo check` updated timestamps but `cargo build` didn't recompile the binary, leaving a stale executable | After editing source files and running `cargo check`, always `touch` the edited file or run `cargo build` directly (not after `cargo check`) to ensure the binary is rebuilt. [2026-03-24] | For CAVLC 8x8 DCT MBs, deblocking used individual 4x4 sub-block NNZ values, but FFmpeg's `fill_filter_caches` (h264_slice.c:2396) overrides these with CBP-based values: if the 8x8 block was coded, ALL sub-blocks get non-zero NNZ for deblocking. [2026-03-24] | Compared wedeo deblock-on vs deblock-off to isolate filter changes, but the diff included MC cascade effects from deblocked references, not just the filter | To isolate the deblock filter itself, compare wedeo vs FFmpeg (both with deblock ON). [2026-03-24] | CABAC 8x8 decode read a coded_block_flag (CBF) that doesn't exist in the bitstream for cat=5, desyncing the CABAC engine for all subsequent symbols. All 20 FRext CABAC files showed 0/N frames matching. | For cat=5 (8x8 luma) in non-chroma-4:4:4, FFmpeg skips the CBF entirely (h264_cabac.c:1859: `cat != 5 || CHROMA444`). The CBP check is the only gate. Always read `decode_cabac_residual_nondc` in FFmpeg to check which cats have CBF and which don't.
[2026-03-24] | Chroma decode used `chroma_qp_index_offset[0]` for both Cb and Cr planes, ignoring `second_chroma_qp_index_offset` (PPS offset[1]). Files with different Cb/Cr QP offsets had V-plane-only diffs. | High profile PPS has `second_chroma_qp_index_offset` for Cr. Always compute per-plane chroma QP: `[CHROMA_QP_TABLE[...offset[0]...], CHROMA_QP_TABLE[...offset[1]...]]`. When diffs are isolated to one chroma plane, suspect QP offset handling.
[2026-03-24] | Spatial direct col_zero_flag optimization was applied unconditionally, but FFmpeg's h264_direct.c:374,405,443 gates it on `!sl->ref_list[1][0].parent->long_ref`. Single-frame B-frame diffs with large Y_max and correct DPB almost always point to direct prediction mode handling.
