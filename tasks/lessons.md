[2026-03-24] | For CAVLC 8x8 DCT MBs, deblocking used individual 4x4 sub-block NNZ values, but FFmpeg's `fill_filter_caches` (h264_slice.c:2396) overrides these with CBP-based values: if the 8x8 block was coded, ALL sub-blocks get non-zero NNZ for deblocking. [2026-03-24] | CABAC 8x8 decode read a coded_block_flag (CBF) that doesn't exist in the bitstream for cat=5, desyncing the CABAC engine for all subsequent symbols. All 20 FRext CABAC files showed 0/N frames matching. | For cat=5 (8x8 luma) in non-chroma-4:4:4, FFmpeg skips the CBF entirely (h264_cabac.c:1859: `cat != 5 || CHROMA444`). The CBP check is the only gate. Always read `decode_cabac_residual_nondc` in FFmpeg to check which cats have CBF and which don't.
[2026-03-24] | Chroma decode used `chroma_qp_index_offset[0]` for both Cb and Cr planes, ignoring `second_chroma_qp_index_offset` (PPS offset[1]). Files with different Cb/Cr QP offsets had V-plane-only diffs. | High profile PPS has `second_chroma_qp_index_offset` for Cr. Always compute per-plane chroma QP: `[CHROMA_QP_TABLE[...offset[0]...], CHROMA_QP_TABLE[...offset[1]...]]`. When diffs are isolated to one chroma plane, suspect QP offset handling.
[2026-03-24] | Spatial direct col_zero_flag optimization was applied unconditionally, but FFmpeg's h264_direct.c:374,405,443 gates it on `!sl->ref_list[1][0].parent->long_ref`. When L1[0] is a long-term reference, col_zero_flag must NOT suppress MVs to zero, because the colocated picture's ref_idx semantics differ for long-term frames. | When porting direct prediction checks, always read ALL conditions in FFmpeg's `if` statement, including `long_ref` checks. Single-frame B-frame diffs with large Y_max and correct DPB almost always point to direct prediction mode handling.
[2026-03-25] | PPS RBSP bit length used `data.len() * 8` which includes the RBSP stop bit padding. FFmpeg's `get_bit_length` (h2645_parse.c:348) strips trailing zero bytes then subtracts `ctz(last_byte) + 1`. This made `bits_left > 0` for Main profile PPS, reading garbage as high-profile extensions, silently failing PPS parse. | Always match FFmpeg's RBSP bit length computation. For any `bits_left` check, the bit length must exclude the stop bit and trailing alignment zeros. Use `rbsp_bit_length()` in pps.rs.

The first IDR has no prior frames to protect. Always test single-IDR files when changing barrier logic.

[2026-03-25] | For MBAFF, used a single save/restore for left-side NeighborContext, but the bottom MB of pair N overwrites the top MB's right edge before pair N+1's top MB reads it.
